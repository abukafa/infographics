1. Kriptografi Modern: Simetris vs Asimetris

Kriptografi bukan sekadar sandi, melainkan fondasi integritas data. Dalam **Simetris**, satu kunci (Shared Secret) digunakan untuk mengunci dan membuka. Sangat cepat namun berisiko jika kunci dicuri saat pengiriman.

**Asimetris (Public-Key)** memecahkan masalah ini dengan dua kunci: **Public Key** untuk mengenkripsi (seperti gembok terbuka yang disebar) dan **Private Key** untuk mendekripsi (kunci fisik yang disimpan rapat oleh pemilik).

Public Key

"Gunakan gembok ini untuk mengirimi saya data."

Private Key

"Hanya kunci saya yang bisa membuka data itu."

Pseudocode: Enkripsi Asimetris 
// Budi ingin mengirim pesan rahasia ke Iwan
PESAN = "Data Penting Perusahaan"
GEMBOK_IWAN = "Kunci_Publik_Iwan"

FUNGSI KirimKeIwan(data, gembok):
    PESAN_TERKUNCI = ASYMM_ENCRYPT(data, gembok)
    KEMBALIKAN PESAN_TERKUNCI

// Di sisi Iwan
FUNGSI TerimaData(terkunci, kunci_fisik):
    PESAN_ASLI = ASYMM_DECRYPT(terkunci, kunci_fisik)
    TAMPILKAN(PESAN_ASLI)

ANALOG WORKSHOP: Simulasi Kertas Kriptografi (Caesar Cipher)

Langkah-Langkah (Enkripsi):

  1. Siapkan kertas, tulis alfabet A-Z.
  2. Tentukan "Kunci" (Angka 1-25). Contoh Kunci = 3.
  3. Geser setiap huruf pada pesan Anda sebanyak 3 posisi ke kanan.
  4. Misal: Huruf 'A' menjadi 'D', 'B' menjadi 'E'.
  5. Tulis pesan rahasia hasil pergeseran tersebut.

Langkah-Langkah (Dekripsi):

  1. Berikan kertas berisi pesan acak kepada rekan Anda.
  2. Berikan "Kunci" (Angka 3) secara rahasia.
  3. Rekan Anda akan menggeser huruf 3 posisi ke KIRI.
  4. Pesan asli akan terbaca kembali.

2. Integritas dengan Hash & Digital Signature

Fungsi Hash (Digital Fingerprint)

Fungsi satu arah (one-way) yang menghasilkan nilai tetap (misal 256-bit). Jika input "A" menghasilkan hash "XYZ", maka input "A." (tambah titik) akan menghasilkan hash yang sama sekali berbeda (Avalanche Effect).

Tanda Tangan Digital (Auth)

Memastikan pengirim adalah asli. Caranya: Pesan di-hash, lalu hasil hash di-enkripsi dengan **Private Key** pengirim. Penerima memverifikasi dengan **Public Key** pengirim.

Pseudocode: Verifikasi Digital 
// Memastikan integritas dan identitas
FUNGSI VerifikasiTandaTangan(dokumen, ttd, kunci_publik):
    // 1. Hitung hash dokumen saat ini
    hash_cek = SHA256(dokumen)
    
    // 2. Dekripsi tanda tangan pengirim
    hash_asli_pengirim = DECRYPT(ttd, kunci_publik)
    
    // 3. Bandingkan
    JIKA hash_cek == hash_asli_pengirim:
        KEMBALIKAN "Data Asli & Sah"
    LAINNYA:
        KEMBALIKAN "Data Palsu/Rusak!"

ANALOG WORKSHOP: Simulasi Logika Hashing (Modulo Hash)

Langkah Simulasi (Membuat Hash):

  1. Tulis sebuah kata (Misal: "Kopi").
  2. Konversi huruf menjadi urutan angka (A=1, B=2, dst).
  3. Misal: K(11), O(15), P(16), I(9).
  4. Jumlahkan semua angka: 11+15+16+9 = 51.
  5. Hitung Sisa Bagi (Modulo) 10: 51 % 10 = **1**.
  6. Angka **1** adalah "Hash" dari kata "Kopi".

Eksperimen (Avalanche Effect):

  • Coba ganti satu huruf saja: "Kopi" menjadi "Kopi." (tambah titik).
  • Titik (simbol) memiliki nilai berbeda.
  • Hasil penjumlahan dan sisa bagi akan berubah drastis.
  • Inilah cara Blockchain tahu jika data diubah walau hanya 1 karakter!

Logika Penambangan (Proof of Work)

Penambang (Miner) harus mencari angka acak (**Nonce**) sehingga hash dari blok tersebut diawali oleh deretan nol tertentu. Ini membutuhkan daya komputasi besar, sehingga mengubah satu blok saja di masa lalu akan memerlukan pengerjaan ulang seluruh rantai.

Pseudocode: Mining Engine 
TARGET = "0000" // Kesulitan jaringan
NONCE = 0

FUNGSI TambangBlok(data_blok):
    SELAMA BENAR:
        HASIL = SHA256(data_blok + NONCE)
        
        JIKA HASIL.DIAWALI(TARGET):
            TAMPILKAN "Blok Sah! Hash: " + HASIL
            BERHENTI
        LAINNYA:
            NONCE = NONCE + 1

Smart Contracts (Kode Adalah Hukum)

Protokol komputer yang memfasilitasi, memverifikasi, atau menegakkan negosiasi kontrak secara otomatis tanpa memerlukan pihak ketiga seperti bank atau notaris.

Pseudocode: Smart Contract Tabungan 
KONTRAK TabunganOtomatis:
    DAFTAR_SALDO = {}

    FUNGSI Setor():
        JUMLAH = Uang_Diterima()
        DAFTAR_SALDO[PENGIRIM] += JUMLAH

    FUNGSI Tarik(nominal):
        // Verifikasi otomatis
        JIKA DAFTAR_SALDO[PENGIRIM] >= nominal:
            KIRIM_KE_DOMPET(PENGIRIM, nominal)
            DAFTAR_SALDO[PENGIRIM] -= nominal
        LAINNYA:
            ERROR "Dana Tidak Cukup"

Fase 3: Konvergensi & Ekonomi Baru

Implementasi dunia nyata dari penggabungan Kriptografi & Blockchain.

Cryptocurrency

Uang digital yang tidak dikontrol bank sentral. Menggunakan Kriptografi untuk otorisasi (Private Key) dan Blockchain untuk mencegah pengeluaran ganda (Double Spending).

Logika Transfer Aset 

// Transfer 1.5 BTC ke Iwan
1. CEK_SALDO(BUDI_ADDR)
2. BUAT_TRANSAKSI(IWAN_ADDR, 1.5)
3. TANDA_TANGANI(TRANS, BUDI_PRIV_KEY)
4. KIRIM_KE_JARINGAN()
5. MINER_VERIFIKASI_DAN_CATAT()

Self-Sovereign Identity

Identitas yang Anda miliki sepenuhnya. Institusi (Pemerintah/Kampus) memberikan sertifikasi (Credential) yang Anda simpan di dompet digital, dan Anda hanya membagikan bukti validitasnya saat dibutuhkan.
